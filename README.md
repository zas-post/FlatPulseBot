# Flat Pulse Bot 🏢🚀

Автономный Telegram-бот на фреймворке **aiogram 3**, предназначенный для непрерывного парсинга и мониторинга рынка недвижимости на площадках **Авито** и **Циан** в режиме реального времени. Бот разработан специально для развертывания в контейнеризированном окружении **Docker** на серверах автоматизации (например, Proxmox).

---

## 🎯 Основной функционал и логика работы

1. **Мультипоточный парсинг в фоне:** Бот запускает два независимых асинхронных воркера (`avito_parser_worker` и `cyan_parser_worker`), которые работают параллельно.
2. **Защита от блокировок (Антифрод):**
   * На старте потоки разводятся во времени на 15 секунд, чтобы избежать пиковой нагрузки на процессор и одновременных запросов с одного IP.
   * Используется библиотека `undetected_chromedriver` со встроенной маскировкой сессий (`cdp_cmd` с подменой `userAgent`).
   * Интервалы проверок динамически колеблются (заданный интервал из настроек плюс/минус 2 минуты случайного времени), имитируя поведение реального человека.
3. **Фильтрация дубликатов:** Все найденные объявления проходят проверку через локальную базу данных **SQLite**. В Telegram отправляются только абсолютно новые предложения, которых еще не было в базе.

## 🛠️ Последние архитектурные доработки и оптимизации

### 1. Безопасная лог-ротация (Защита диска сервера)
Для исключения риска переполнения дискового пространства сервера логированием ошибок, стандартный вывод переведен на профессиональный хэндлер `RotatingFileHandler`:
* **Файл логирования ошибок:** `errors.log`
* **Максимальный размер одного файла:** 5 МБ (`maxBytes=5*1024*1024`)
* **Количество архивных копий:** 3 (`backupCount=3`)
При достижении лимита бот автоматически переименовывает старый файл и начинает запись в новый.

### 2. Продвинутая отказоустойчивость (Self-Healing логика)
Парсеры защищены от кратковременных сетевых сбоев и зависаний процессов Chromium. При возникновении ошибки типа `Remote end closed connection without response` воркер фиксирует сбой, делает паузу 10-12 секунд и осуществляет повторную попытку (`retry`). Контейнер не падает и продолжает стабильную работу.

### 3. Чистая Docker-сборка (`.dockerignore`)
Добавлен файл конфигурации исключений для Docker. При сборке образа (`docker compose build`) внутрь контейнера больше не копируются:
* Локальные файлы баз данных (`*.db`, `*.sqlite3`).
* Временные файлы кэша Python (`__pycache__`, `*.pyc`).
* Локальные приватные логи и файлы окружения (`.env`, `.venv`).
Это делает образ легковесным, а данные — изолированными.

## 📱 Юзабилити и интерактивный интерфейс сообщений

В боте реализована удобная система управления входящим потоком объявлений с помощью инлайн-кнопок:

* **⭐️ В семейный чат:** Мгновенно пересылает выбранную карточку в общий семейный чат с сохранением всего красивого HTML-форматирования (но удаляет технические кнопки управления, чтобы не засорять чат).
* **❌ Удалить (Архивация карточки):** Вместо резкого удаления сообщения (которое ломает ленту чата), карточка «сворачивается» в одну компактную, ультра-читаемую строку:
  > `⚪️ Объявление скрыто (Источник / Комнатность и площадь / Стоимость)`
* **↩️ Восстановить текст:** Если кнопка скрытия была нажата случайно, рядом со скрытой строкой остается кнопка отмены. При её нажатии бот «на лету» воссоздает исходную структуру сообщения и возвращает все управляющие кнопки назад.
* **🔗 Ссылка:** Кнопка быстрого перехода на сайт Авито/Циан доступна даже у свернутого (скрытого) объявления.
